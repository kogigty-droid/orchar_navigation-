## 一、本发明要解决的技术问题及所属技术领域

### 技术领域

本发明属于无人车与无人机协同控制技术领域，具体涉及一种利用无人机与地面车辆实时位置互感实现动态路径生成与跟踪的车机协同方法、系统及设备。

### 要解决的技术问题

在车载无人机协同作业场景中，地面车辆需要根据无人机的实时位置动态调整行驶路径，以实现车辆到达无人机下方或侧方进行对接、回收或补给。现有技术中，离线规划方法（如 CNxxxxx 车机协同林业监测联合优化方法）依赖预计算的路网距离矩阵与固定停靠点方案，无法应对无人机位置实时变化的动态场景；而简单的"车辆直接朝无人机经纬度坐标行驶"的方法则忽略了车辆运动学约束、坐标系对齐和路径可执行性问题。本发明要解决的核心问题是：如何利用无人机与车辆的双端实时位置信息，在统一坐标系下动态生成可执行路径，并使车辆稳定跟踪该路径到达无人机附近。

## 二、背景技术及现有技术方案

### 2.1 车机协同林业监测联合优化方法（已公开专利）

该专利（见摘要/权利要求）提出一种车机协同林业监测的离线联合优化方法，包括：对监测区域网格化并筛选候选停靠点；采用聚类分区策略划分子区域监测单元；基于路网距离矩阵用贪心最近邻算法构造车辆访问顺序；采用遗传算法优化各子区域无人机航向角；构建帕累托解集；基于 Fix-and-Optimize 框架交替优化停靠点组合与子区域重划分。该方法输出一组覆盖率与总作业时间权衡的非支配调度方案。

该方法的核心特征为：
- 依赖预构建的路网距离矩阵
- 停靠点为预设固定位置
- 车辆路径在任务开始前一次性计算完成
- 无人机按牛耕式航线在子区域内飞行，航向角离线优化

### 2.2 车机协同自主跟踪与降落方法（CN114326766A，中科院深圳先进院）

该方法通过车载相机检测无人机上的二维码标记，获取二维码在相机坐标系下的位姿，根据无人机抖动情况自适应调节 PID 参数，协同输出无人机航向速度和车辆行驶速度，实现无人机降落至移动车辆平台。该方法依赖视觉标记，受限于相机视场和光照条件。

### 2.3 车载无人机精准定位控制方法及装置（CN120803008A，2025）

该方法获取无人机实时位置进行 RTK 差分处理，结合车载端加速度矢量、姿态矩阵和时间差，计算无人机实际精准位置，实现无人机精准降落回车载储运箱。该方法侧重于无人机与车辆之间的精确相对定位计算。

### 2.4 简单的直接追踪方法

工程实践中存在一种简单做法：车辆直接使用无人机的经纬度坐标作为目标点，计算两者经纬度差值后直接朝该方向行驶。该方法忽视了几个关键问题：双端坐标系未统一（WGS84 → 局部投影坐标系）；车辆初始航向角未校准；未考虑车辆最小转弯半径、道路边界和障碍物约束。

## 三、现有技术的缺点

（1）**离线规划方法不适应动态场景**：林业监测专利等方法假设任务前已知所有停靠点和路网，车辆按预定顺序访问固定停靠点。当无人机因任务需求（如避障、目标识别、风扰）偏离预定航迹时，车辆无法实时调整汇合位置。

（2）**视觉跟踪方法受环境限制**：二维码/视觉标记方案（CN114326766A）依赖相机视场、光照条件和标记可见性，在遮挡、强光、夜间或远距离场景下可靠性下降。

（3）**简单追踪方法忽略车辆运动约束**：直接朝无人机经纬度行驶的做法未进行坐标系统一和航向角校准，且未考虑车辆最小转弯半径、道路边界和障碍物约束，容易产生不可执行的路径（如穿越不可通行区域、急转弯超出车辆能力）。

（4）**缺乏完整的异常处理机制**：现有方法通常未系统性地处理 RTK 失锁、通信中断、目标点不可达等异常工况下的安全降级策略。

---

## 四、本发明的目的

本发明的目的在于提供一种基于实时位置互感的车机动态协同方法，该方法以地面车辆为执行核心，以无人机为任务感知与定位辅助端，通过以下手段解决上述问题：

- 利用无线通信模块获取无人机实时位置与任务信息，结合车载 RTK 定位，将双端位置统一转换至同一局部坐标系；
- 将无人机实时位置或任务位置转化为车辆可到达的汇合点（而非直接追无人机坐标）；
- 在统一坐标系下进行航向角校准后，在线生成满足车辆运动学约束的局部目标路径；
- 通过路径跟踪控制使车辆稳定到达汇合点（无人机下方或侧方）；
- 具备完整的异常检测与安全降级机制（RTK 失锁、通信超时、路径不可执行）。

## 五、本发明技术方案的详细阐述

### 5.1 系统总体架构

本发明的系统由三部分组成：

- **无人机端**：包括飞控模块、RTK/GNSS 定位模块、无线通信模块。无人机端持续获取自身位置（经纬度、高度）、飞行状态（速度、航向）和任务信息（当前悬停点、目标作业点、投放点等），通过无线通信模块发送至地面车辆端。
- **地面车辆端**：包括车载计算单元、RTK/GNSS 定位模块、可选视觉/IMU 辅助定位模块、无线通信接收模块、底盘运动控制模块。车载计算单元负责：双端位置信息接收与校验、坐标统一转换、航向角估计、汇合点生成、局部路径规划、路径跟踪控制、异常检测与安全降级。
- **通信链路**：无人机与车辆之间通过无线通信模块（如数传电台、Wi-Fi、4G/5G、LoRa 等）进行数据传输。

### 5.2 数据采集与状态判断

车辆端实时采集以下数据：
- 车辆 RTK 经纬度、高程、定位质量标志、时间戳
- 无人机 RTK 经纬度、高程、定位质量标志、时间戳
- 无人机任务信息（当前悬停点、目标作业点、投放点等）
- 车辆当前速度、角速度、控制模式（自动/手动）
- 可选：IMU 姿态数据、视觉感知数据

车辆端对定位数据进行有效性判断：
- 当 RTK 为固定解（Fixed）状态时，判定定位信息有效
- 当 RTK 由固定解降级为浮点解、差分伪距解或单点解时，判定定位异常
- 当无人机数据在预设时间窗口内未更新时，判定通信或无人机定位异常

### 5.3 统一坐标系的建立

无人机和车辆获取的原始 RTK 数据通常为 WGS84 经纬度坐标，无法直接用于小范围路径规划。本方法将其统一转换至同一局部坐标系。

局部坐标系可采用 ENU 坐标系（东北-天坐标系），以作业区域内一参考点 P₀ = (lat₀, lon₀, h₀) 为原点。参考点的选取方式包括：作业区域入口点、RTK 基准站点、首次作业起始点或车辆启动时位置。

转换步骤：
1. 将参考点 P₀、无人机位置 Pᵤ = (latᵤ, lonᵤ, hᵤ) 和车辆位置 Pc = (latc, lonc, hc) 由经纬度坐标转换为空间直角坐标（ECEF）
2. 根据参考点经纬度构造旋转矩阵，将空间直角坐标转换为 ENU 坐标
3. 得到无人机局部坐标 Pₑᵤ = (xᵤ, yᵤ, zᵤ) 和车辆局部坐标 Pₑc = (xc, yc, zc)

无人机相对车辆的局部位置向量为：
```
P_rel = Pₑᵤ - Pₑc = (xᵤ - xc, yᵤ - yc, zᵤ - zc)
```

在小范围作业场景中，也可采用经纬度线性投影或平面近似方式转换，但应确保无人机与车辆使用同一原点与坐标方向。

### 5.4 车辆航向角估计与坐标系旋转

为避免初始航向角不准确导致路径偏斜，本方法对车辆航向角进行估计和校准。

航向角 ψ_car 定义为车辆前进方向与地理北向的夹角。当车辆行驶速度高于预设阈值且 RTK 定位有效时，利用相邻时刻位置计算航向角：
```
ψ_car = atan2(yc(t) - yc(t-1), xc(t) - xc(t-1))
```
当车辆静止或速度过低时，采用 IMU 或双天线 RTK 提供的航向角作为补充。

获得航向角后，将无人机在局部坐标系中的相对位置转换到车辆本体坐标系：
```
P_body = R(-ψ_car) · P_rel
```
其中 R(-ψ_car) 为二维旋转矩阵。P_body = (xb, yb) 表示无人机在车辆"前-左"方向上的相对位置，使后续路径规划以车辆实际前进方向为基准。

### 5.5 汇合点的生成

本方法的核心创新之一是：**不直接将无人机实时位置作为车辆目标点，而是将其转化为车辆可到达的汇合点**。

汇合点 P_goal 的计算公式为：
```
P_goal = Pₑᵤ + D_offset
```
其中 D_offset 为偏移向量，根据以下因素确定：
- 无人机的朝向与任务类型（如悬停、向前飞行）
- 作业区域通行方向
- 作业安全距离（如避免车辆靠近无人机下方旋翼）
- 对接/装载/投放操作的相对位置

例如，为使车辆从无人机侧方安全接近，可将汇合点设在无人机投影位置前方或侧方一定距离处。D_offset 由系统预设或根据任务类型动态配置。

### 5.6 局部路径生成

车辆根据当前本体坐标系下的汇合点 P_body，结合车辆运动约束，生成局部目标路径。

路径生成需考虑以下约束：
- 车辆最小转弯半径
- 车辆最大速度与最大加速度
- 道路边界与障碍物边界
- 汇合对接距离要求

路径形式可采用：
- 直线-圆弧组合路径（适用于小偏角近距场景）
- 三次多项式螺旋线
- B 样条曲线
- Dubins 路径或 Reeds-Shepp 路径
- 在安全走廊内通过二次规划（QP）生成平滑轨迹

其中推荐采用**安全走廊 + QP 平滑路径**方法：
1. 在局部规划范围内，根据道路边界、障碍物边界和车辆尺寸构造安全走廊（可通行约束空间）
2. 在安全走廊内，通过二次规划求解平滑轨迹，优化目标包括：最小化对汇合点方向的偏差、最小化对通行道路中心线的偏离、最小化路径曲率、最小化曲率变化率、最小化横向加速度或 jerk

路径的起点约束为车辆当前位置和当前姿态，终点约束为汇合点位置及期望到达方向。若路径曲率超出车辆执行能力范围，系统自动降低目标速度。

### 5.7 目标平滑与路径稳定性

为避免无人机位置短时间突变导致车辆路径频繁震荡，本方法对汇合点进行时域平滑处理：

- 对新接收的无人机位置，采用低通滤波或移动平均抑制高频噪声
- 当无人机在小范围内漂移且当前路径仍可执行时，车辆维持当前路径行驶
- 设置汇合点更新阈值：只有当新汇合点与当前目标点距离超过阈值、或当前路径剩余距离低于阈值时，才触发路径重规划
- 当无人机飞抵新悬停点或任务点导致汇合点大幅变化时，车辆先减速再切换新目标

### 5.8 路径跟踪控制

车辆对生成的局部目标路径执行路径跟踪控制。

路径跟踪算法可选用：
- Pure Pursuit（纯追踪）：通过动态预瞄距离在前方路径上选取跟踪目标点
- Stanley 控制器：结合横向偏差和航向偏差计算转向角
- 模型预测控制（MPC）：在预测时域内优化车辆控制输入
- LQR：基于线性二次型最优控制

以 Pure Pursuit 为例，动态预瞄距离公式为：
```
Ld = L₀ + kv·v + ke·|ey| + kκ·|κ|
```
其中 Ld 为预瞄距离，L₀ 为基础预瞄距离，v 为车辆速度，ey 为横向跟踪偏差，κ 为局部路径曲率，kv、ke、kκ 为可调系数。速度较高或路径平直时适当增大预瞄距离，路径曲率大时适当降低速度并减小预瞄距离。

对于履带式底盘，路径跟踪输出的线速度 v_cmd 和角速度 w_cmd 转换为左右履带目标速度：
```
v_left  = v_cmd - w_cmd × B / 2
v_right = v_cmd + w_cmd × B / 2
```
其中 B 为左右履带中心距。

### 5.9 视觉辅助定位与路径源切换（可选）

当车辆同时配备视觉感知模块（如前方相机检测道路边界/作物行）时，可将其作为辅助定位与路径信息来源：

- 视觉模块检测道路左右边界或作物行，计算视觉路径和置信度
- 视觉置信度综合考虑：左右边界检测的对称性、边界连续性、边界线变化率和图像有效信息占比
- 当视觉路径置信度高于阈值时，可将视觉路径作为前向参考，与 RTK 协同路径做融合或偏差补偿
- 当视觉置信度低于阈值（如边界不对称、一侧缺失、光照剧变）时，停止使用视觉路径，完全切换为基于 RTK 的协同路径

视觉模块在本方法中为辅助角色，核心路径来源仍为基于双端 RTK 位置的协同路径。

### 5.10 异常检测与安全降级

本方法具备完整的异常检测和安全降级机制，涵盖以下异常类型：

| 异常类型 | 检测条件 | 处理策略 |
|---------|---------|---------|
| RTK 定位异常 | RTK 固定解丢失，定位质量低于阈值 | 利用最近有效航向角维持短时行驶；若持续超过阈值则减速停车，切换手动模式 |
| 无人机通信超时 | 连续未收到无人机数据超过预设时间窗口 | 车辆沿最后有效路径行驶一段距离；若超时持续则减速停车 |
| 路径不可执行 | 生成路径的曲率超出车辆能力范围，或跟踪偏差超过阈值 | 降低目标速度重新尝试；仍不可行则停车并触发重规划 |
| 汇合点不可达 | 汇合点落入不可通行区域或超出安全走廊 | 回退至上一有效汇合点，或重新搜索最近可达点 |
| 人工接管 | 操作员切换至手动模式 | 底盘控制器立即停止执行自动控制指令，切换为遥控器指令，且遥控器指令享有最高优先级 |

### 5.11 自动/手动模式切换

车辆具备自动模式和手动模式，且手动模式始终具有更高优先级。

- **手动模式**：遥控器信号直接控制底盘运动，车辆操作员可随时接管
- **自动模式**：自动模式仅在以下条件同时满足时才被激活：
  - 遥控器自动模式开关有效
  - RTK 定位为固定解且精度满足预设要求
  - 无人机位置数据更新正常且未超出通信超时阈值
  - 目标路径曲率不超过底盘执行能力范围
  - 目标速度和转向角速度在底盘执行范围内
- 以上任一条件不满足时，系统自动降级为手动模式并减速或停车

### 5.12 总体方法流程

```
S1. 在车辆端部署计算单元、RTK定位模块、无线通信模块和底盘控制模块；
S2. 无人机端部署飞控模块、RTK/GNSS定位模块和无线通信模块；
S3. 车辆启动后初始化各模块，等待RTK达到固定解；
S4. 车辆通过无线通信模块接收无人机实时位置和任务信息，同时获取车辆自身RTK定位信息；
S5. 对定位数据进行有效性判断（RTK状态、通信时效性）；
S6. 将无人机和车辆的经纬度坐标统一转换至同一局部坐标系（如ENU坐标系）；
S7. 估计车辆当前航向角，将无人机相对位置转换至车辆本体坐标系；
S8. 根据无人机位置/任务位置计算车辆可到达的汇合点；
S9. 结合车辆当前位姿、汇合点、运动约束和局部障碍信息，生成局部可达路径；
S10. 对汇合点进行时域平滑，避免路径频繁重规划；
S11. 路径跟踪算法输出线速度和角速度指令，通过自动模式控制底盘运动；
S12. 当到达汇合点对接范围、人工切换手动模式、或检测到定位/通信异常时，执行对接、减速停车、重规划或人工接管。
```

## 六、本发明的关键点与欲保护点

### 关键点一：无人机位置 → 汇合点的转化机制

本发明不同于直接追无人机坐标的方案，而是将无人机实时位置（或任务位置）结合车辆可达性约束转化为汇合点。现有技术中，林业监测专利的停靠点为离线预设、固定不变；简单追点方法直接以经纬度差值驱动车辆。本发明在无人机位置和车辆目标点之间插入"汇合点生成"环节，综合考虑无人机朝向、任务类型、安全距离和车辆运动约束。

### 关键点二：双端坐标统一 + 航向角校准 + 车体坐标系旋转

现有简单方法直接在经纬度层面计算偏差，未进行坐标系统一和航向角校准，导致初始路径偏斜。本发明先将双端 WGS84 坐标统一至同一 ENU 局部坐标系，再估计车辆航向角后进行车体坐标系旋转，使路径规划以车辆实际前进方向为基准，消除初始偏斜。

### 关键点三：安全走廊 + 约束路径生成 + 路径跟踪的闭环

现有离线规划方法不涉及实时路径跟踪控制。本发明在汇合点确定后，在安全走廊内生成满足车辆运动约束的局部路径，并通过路径跟踪算法形成控制闭环，确保路径可执行、跟踪稳定。

### 关键点四：系统化的异常检测与安全降级

现有技术通常未完整覆盖 RTK 失锁、通信中断、路径不可执行等异常工况。本发明建立了从检测条件到处理策略的完整异常处理链条，且手动模式始终享有最高优先级。

## 七、与现有技术相比较，本发明的优点

（1）**动态实时性好**：本发明利用无线通信模块实时获取无人机位置，车辆在线生成路径并跟踪，无需预知无人机航线或预设固定停靠点。相比林业监测专利的离线规划框架，更适合无人机位置动态变化的场景。

（2）**路径可执行性高**：本发明通过坐标统一、航向角校准、汇合点转化和运动约束路径生成，确保输出路径满足车辆转弯能力和通行约束。相比直接追点方法，避免了路径不可执行或跟踪偏差过大的问题。

（3）**环境适应性强**：本发明以双端 RTK 定位为核心、视觉感知为可选辅助，不依赖二维码标记或特定光照条件。相比纯视觉跟踪方案，在遮挡、远距离、夜间等场景下仍可正常工作。

（4）**安全性好**：本发明具备从 RTK 失锁、通信中断到人工接管的完整降级链路，手动模式始终享有最高优先级，保障了作业安全。

## 九、替代方案

（1）**无人机定位方式**：无人机端可不限于 RTK，也可采用 GNSS 单点定位、双频 GNSS + RTK、RTK + IMU 组合、视觉惯性里程计或激光雷达定位，只要能够提供无人机的位置信息即可。

（2）**通信方式**：无线通信不限于数传电台/MAVLink，也可采用 4G/5G 蜂窝网络、Wi-Fi、LoRa、ZigBee 或专用无线图传链路。

（3）**局部坐标系**：不限于 ENU 坐标系，也可采用以行驶方向为纵轴、以横向为横轴的 Frenet 坐标系，或以车辆当前位置为原点的局部平面坐标系。

（4）**航向角获取**：不限于 RTK 位置差分，也可采用 IMU 航向角、双天线 RTK 定向、磁力计或里程计。

（5）**路径生成方法**：不限于安全走廊+QP，也可采用三次多项式、Bezier 曲线、Dubins/Reeds-Shepp 路径、Hybrid A*、或基于采样的规划器（RRT 及其变体）。

（6）**路径跟踪算法**：不限于 Pure Pursuit，也可采用 Stanley、LQR、MPC、滑模控制或自适应控制。

（7）**底盘类型**：不限于履带式底盘，也可适用于轮式差速底盘、阿克曼转向底盘或铰接式转向底盘，只需将控制指令适配至相应底盘运动学模型。

（8）**视觉模块**：视觉感知可作为辅助信息源，也可替换为激光雷达道路检测、毫米波雷达边界检测或预先标定的道路边界。
